[001] Overview

davidstarOS is an Arch-based Linux distribution with a strong focus on simplicity, transparency and user control. The project was created by blck and ninja who also act as its sole developers and BDFLs [0].

The distribution is designed to be maintainable by a small team and to this end has no infrastructure, backend or monetary running costs. Further, all installations of davidstarOS contain everything needed to continue its development.

The user is provided with three things.

  • The Official Repositories — An extensible base including a fully X11-less Wayland environment, modern web browser, media player and everything needed for this system to rebuild (and update) itself (in under 150 packages).
  • The Package Management System — A pleasant to work with and concise package format powered by the Arch package manager (pacman).
  • The Freedom To Go It Alone — Users can choose to maintain their system themselves thus freeing them from all external influence. The system then belongs to the user and they become totally self-reliant.

[002] The Official Repositories

The problem with monolithic repositories is that they have no clear end. They have infinite growth possibility, very broad scope and rely on volunteer effort to keep everything fresh. They become harder to maintain over time and unless their growth is checked, will eventually become unsustainable.

davidstarOS's repositories are instead designed to be small and understandable. Instead of attempting to package the world they act as an extensible base which users can build upon to suit their needs.

To best illustrate, here is the entire thing (145 components) (2026).

The distribution explicitly excludes systemd, pulseaudio, dbus, polkit, and all desktop environments. This software is either too complex or otherwise out of scope.

[003] Package System

The system is based around the Arch Linux package management system using pacman as the primary package manager. Packages are distributed as binary archives (.pkg.tar.zst) and installed via the standard pacman commands.

The distribution provides a curated set of packages from the Arch Linux repositories, with additional custom packages built and maintained by the davidstarOS team.

# Basic pacman commands pacman -Syu # Update system pacman -S [package] # Install package pacman -R [package] # Remove package pacman -Q # List installed packages pacman -Ss [query] # Search for packages

Each package contains the compiled binaries, configuration files and metadata needed for installation. The package format is standardized across the Arch Linux ecosystem.

Further reading: @/wiki/package-system

[004] Package Manager

The package manager (pacman) is a powerful and efficient package management system used by Arch Linux and its derivatives. It is written in C and designed for speed and simplicity.

Pacman uses a simple command-line interface with a consistent syntax. The main operations are:

# pacman [operation] [options] [packages] pacman -S # Install package(s) pacman -R # Remove package(s) pacman -U # Upgrade/install from local file pacman -Q # Query package database pacman -Sy # Update repository databases pacman -Syu # Update the entire system pacman -Ss # Search for packages pacman -Si # Display package information pacman -Sc # Remove unused packages

Additionally, the distribution includes helper scripts for common tasks and a custom repository for davidstarOS-specific packages.

Further reading: @/wiki/package-manager

[005] Software Distribution Model

The distribution is "rolling release" [1] (install once, update perpetually) with binary packages provided by the Arch Linux infrastructure. Updates to the distribution are distributed via the standard pacman package manager.

This model gives the user access to the vast Arch Linux repository while also maintaining a curated base system with a focus on simplicity and minimalism.
